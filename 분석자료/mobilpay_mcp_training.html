01 — Overview

프로젝트 개요

MOBILPAY(KG모빌리언스) 결제연동 REST API 매뉴얼을 MCP(Model Context Protocol) 서버로 구축하여, 가맹점 개발자가 AI 코딩 도구에서 자연어로 정확한 결제 연동 코드를 생성할 수 있는 환경을 만든 프로젝트입니다.

🎯

목표

한국 PG 업계 최초 MCP 도입. 가맹점 결제연동 기간을 기존 5~10일에서 1일 이내로 단축

📊

벤치마크

토스페이먼츠 MCP 서버 구현 방식을 철저히 분석하여 동일 수준의 아키텍처로 구축

⚡

핵심 성과

Claude 하나로 기획안 → 18개 API 문서 → MCP 서버 개발 → NPM 배포 → Skills → 개발자 가이드까지 완성

02 — Why MCP

왜 MCP가 필요한가

AI 코딩 도구가 결제 연동 코드를 생성할 때 MCP 없이는 치명적 오류가 발생합니다. 토스페이먼츠의 내부 실험 결과를 벤치마킹하여 MCP 도입의 필요성을 확인했습니다.

MCP 없이는 AI가 생성한 결제 코드에 보안 취약점이 포함되고, 잘못된 API 사용으로 결제 장애가 발생할 수 있습니다.
⚠️ 핵심 문제: AI 환각(Hallucination)

AI 코딩 도구가 MCP 없이 결제 코드를 생성하면 skey가 클라이언트에 노출되거나, 존재하지 않는 API 엔드포인트를 호출하거나, 필수 파라미터를 누락하는 등 심각한 오류가 발생합니다. MCP는 AI에게 정확한 API 명세를 실시간으로 제공하여 이 문제를 근본적으로 해결합니다.

채팅창은 기획/문서 생성에 최적. Claude Code는 실제 개발/테스트/배포에 최적. Cowork은 파일 생성 + Eval 벤치마크에 최적. 하나의 도구에서 모든 걸 하려 하지 않고, 각 도구의 장점에 맞춰 작업을 분배한 것이 핵심 전략이었습니다.

06 — Architecture

기술 아키텍처

MOBILPAY MCP 서버의 내부 구조와 데이터 흐름을 상세히 살펴봅니다.

MOBILPAY MCP 서버 전체 데이터 흐름: AI 도구 → STDIO → MCP 서버 → BM25 검색 → MD 문서 → 정확한 코드 생성

BM25 검색 엔진 핵심 코드

src/search/bm25.ts (핵심 로직)
// BM25 스코어 계산 — 토스페이먼츠와 동일한 파라미터
const k1 = 1.2;   // 용어 빈도 포화 상수
const b  = 0.75;  // 문서 길이 정규화 상수

function calculateBM25(tf: number, df: number, docLen: number, avgDl: number, N: number): number {
  const idf = Math.log((N - df + 0.5) / (df + 0.5) + 1);
  const tfNorm = (tf * (k1 + 1)) / (tf + k1 * (1 - b + b * (docLen / avgDl)));
  return idf * tfNorm;
}

// Chunk Window: 최상위 청크 + 인접 청크(같은 문서만!) 반환
function expandChunkWindow(chunk: Chunk, allChunks: Chunk[]): Chunk[] {
  return allChunks.filter(c =>
    c.source === chunk.source &&  // ← cross-document bleed 방지
    Math.abs(c.index - chunk.index) <= 1
  );
}

10 — Key Takeaway

핵심 교훈: AI를 업무에 활용하는 방법

이 프로젝트를 통해 얻은 실전 인사이트를 공유합니다.

🎯

1. 도구별 역할을 나눠라

채팅창은 전략·기획·문서 생성, Claude Code는 개발·테스트·배포, Cowork은 평가·패키징. 하나의 도구에서 모든 걸 하려 하면 비효율적입니다.

📋

2. 기획을 먼저 잡아라

벤치마킹 분석 → 기획안 → 문서 생성 → 개발 순서로 진행. AI에게 명확한 설계를 먼저 주면 코드 품질이 크게 향상됩니다.

🔄

3. 세션 간 컨텍스트를 이어라

정리본을 작성하여 다음 세션에 넘기면 컨텍스트 단절 없이 연속적으로 작업 가능. 이 교육자료도 3개 정리본을 통합하여 작성한 것입니다.

📏

4. 정량 효과를 반드시 측정하라

Eval 벤치마크로 +43% 정확도 향상을 실측. 임원 보고 시 "체감" 아닌 "수치"로 보여주는 것이 핵심입니다. 베타 테스트 후 실측 데이터 수집이 다음 과제입니다.

🤖

5. AI의 자가 진단을 활용하라

Claude Code는 빌드 에러를 스스로 분석하고 수정합니다. 모든 버그를 사람이 잡으려 하지 말고, AI가 테스트→에러 분석→수정하는 루프를 돌게 하세요.

🚀

6. 트렌드를 선점하라

MCP → Skills → CLI 순서로 AI 개발 트렌드가 확산 중. 빠르게 도입하면 "업계 최초"라는 포지셔닝으로 브랜드 가치를 높일 수 있습니다.
