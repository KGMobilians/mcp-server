## 5. 핵심 인사이트

### AI가 특히 효과적이었던 영역

| 영역 | 이유 |
|------|------|
| **반복적 문서 변환** | 원본 매뉴얼 → 27개 구조화된 MD 파일 변환은 단순 반복 작업으로 AI에 최적 |
| **보일러플레이트 생성** | package.json, tsconfig, 테스트 인프라 등 프로젝트 초기 설정 자동화 |
| **알고리즘 구현** | BM25, Chunker 등 잘 알려진 알고리즘은 AI가 정확하게 구현 |
| **테스트 코드 생성** | 구현 코드와 동시에 테스트를 생성하여 별도 테스트 작성 공수 제거 |
| **벤치마킹 & 리서치** | 경쟁사(토스페이먼츠) 분석, MCP 프로토콜 학습을 수 분 내 완료 |

### 사람이 여전히 필수적인 영역

| 영역 | 이유 |
|------|------|
| **도메인 판단** | 결제 보안 요구사항, skey 보호 정책 등은 사람의 도메인 지식이 필요 |
| **품질 검증** | AI 산출물의 정확성 확인, 버그 발견(재귀 읽기, cross-document bleed) |
| **의사결정** | Tool 구성, 아키텍처 선택, 배포 전략 등 최종 판단 |
| **이해관계자 소통** | 기획안 검토, 개발팀 협업, 가이드 페이지 요구사항 조율 |
