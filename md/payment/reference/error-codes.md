# 에러코드표

MOBILPAY API 사용 시 발생할 수 있는 에러코드입니다. 모든 API의 응답에서 `code` 필드로 결과를 확인합니다.

## 공통 응답 코드

| 코드 | 설명 |
|------|------|
| `0000` | 정상처리 (성공) |

## API 에러 코드

| 코드 | 메시지 |
|------|--------|
| `6101` | 유효하지 않은 parameter |
| `6102` | 유효하지 않은 hmac |

> **참고**: 에러 발생 시 `code`와 `message` 필드를 함께 확인하세요. `message`에 상세한 에러 원인이 포함됩니다.

## 에러 대응 가이드

### 6101 (유효하지 않은 parameter)

- 필수 파라미터가 누락되었는지 확인
- 파라미터의 데이터 타입과 최대 길이가 명세서와 일치하는지 확인
- JSON 형식이 올바른지 확인

### 6102 (유효하지 않은 hmac)

- `skey` 값이 정확한지 확인
- HMAC 메시지 구성 순서 확인 (거래등록: `amount+ok_url+trade_id+time_stamp`, 취소: `amount+pay_token`)
- `time_stamp` 값이 정확한지 확인
- UTF-8 인코딩으로 처리하고 있는지 확인
