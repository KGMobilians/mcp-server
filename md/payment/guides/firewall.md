# 방화벽 설정

MOBILPAY 결제 서비스를 이용하기 위해, 가맹점 서버에서 방화벽을 사용하는 경우 당사 결제서버 IP를 허용해야 정상적인 이용이 가능합니다.

| NOTI 서버 | - | 175.158.12.2 | 443 |

| 서버 종류 | 도메인 | IP | PORT |

## 방화벽 설정 항목 요약

가맹점 서버에서 아래 IP/PORT에 대한 아웃바운드(outbound) 및 인바운드(inbound) 통신을 허용해야 합니다.

### 아웃바운드 (가맹점 → MOBILPAY)

거래등록, 결제승인, 결제취소 등 API 호출 시 필요합니다.

| 대상 IP | PORT | 용도 |
|---------|------|------|
| 175.158.12.152 | 443 | 테스트 API 호출 |
| 175.158.12.133 | 443 | 운영 API 호출 (메인) |
| 116.127.121.151 | 443 | 운영 API 호출 (백업) |

### 인바운드 (MOBILPAY → 가맹점)

`noti_url` 결제 결과 통보 수신 시 필요합니다.

| 출발 IP | PORT | 용도 |
|---------|------|------|
| 175.158.12.2 | 443 | 테스트 NOTI 수신 |
| 175.158.12.1 | 443 | 운영 NOTI 수신 (메인) |
| 116.127.121.132 | 443 | 운영 NOTI 수신 (백업) |

## noti_url IP 검증용 목록

`noti_url` 처리 시 호출 IP가 MOBILPAY 결제서버인지 검증해야 합니다.

```javascript
// noti_url IP 검증에 사용할 IP 목록
const MOBILPAY_NOTI_IPS = [
  '175.158.12.1',   // 메인 NOTI 서버
  '175.158.12.2',   // 테스트 NOTI 서버
  '116.127.121.132' // 백업 NOTI 서버
];
```

## 프로토콜

모든 통신은 **HTTPS (443 포트)**를 사용합니다. HTTP는 지원하지 않습니다.
