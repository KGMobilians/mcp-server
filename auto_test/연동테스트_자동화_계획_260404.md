# MCP 서버 연동테스트 자동화 계획

> 작성일: 2026-04-04
> 목적: IT기획운영팀의 수동 QA 테스트를 자동화하여 테스트 → 보완 → 재테스트 사이클을 Claude Code가 직접 수행

---

## 1. 배경

MCP + Skills 개발 완료 후, 가맹점 개발자 입장에서의 품질 검증을 IT기획운영팀이 수동으로 진행 중.
이를 자동 테스트 체계로 전환하여 반복 가능하고 빠른 QA 사이클을 구축한다.

### 현재 상태
- **기존 테스트**: 유닛 테스트 20개 (HMAC 6, BM25 7, Chunker 7) — 내부 로직만 커버
- **미테스트 영역**: 6개 MCP 도구 핸들러, API 이름 해석, 코드 예제 추출, 개발자 시나리오
- **CI/CD**: 없음
- **테스트 프레임워크**: Vitest 3.2.4

---

## 2. 테스트 4계층 구조

| 계층 | 목적 | 범위 |
|------|------|------|
| **Layer 1** 도구 핸들러 | 각 MCP 도구의 입출력 정확성 | 이름 해석, 검색 정확도, 코드 추출, 에러 처리 |
| **Layer 2** E2E 프로토콜 | MCP 프로토콜 레벨 동작 검증 | 도구 목록, 스키마, 타입 에러, 필수값 누락 |
| **Layer 3** 시나리오 QA | 가맹점 개발자 실사용 시나리오 | 결제 연동, HMAC 구현, 플로우 이해, 에러코드 확인 |
| **Layer 4** 보안 규칙 | 보안 컴플라이언스 자동 검증 | skey/MAC_KEY 보호, 서버사이드 처리, 멱등성 안내 |

---

## 3. 기술 구현 방식

### 핵심: InMemoryTransport 기반 테스트 클라이언트

MCP SDK의 `InMemoryTransport.createLinkedPair()`를 활용하여 서버를 in-process로 띄우고 `client.callTool()`로 직접 도구 호출. 별도 프로세스 없이 빠르고 안정적인 테스트 가능.

```
[Test Code] → Client → InMemoryTransport → McpServer → Tool Handler → 응답 검증
```

- `buildIndex()` 1회 호출로 27개 문서 인덱싱
- 6개 도구 등록 (index.ts 로직 재현)
- `callTool(name, args)` 헬퍼로 간편한 도구 호출

## 7. 테스트 → 보완 자동 사이클

Claude Code가 직접 수행하는 반복 워크플로우:

```
┌─────────────────────────────────────────┐
│  1. npm run test:qa 실행                 │
│  2. 실패 항목 분석 → 원인 분류            │
│  3. 원인별 수정 적용                      │
│  4. npm run test:qa 재실행               │
│  5. npm run test 전체 회귀 테스트         │
│  6. 전체 통과 시 완료 / 실패 시 2로 복귀   │
└─────────────────────────────────────────┘
```

### 실패 원인별 수정 매핑

| 실패 유형 | 수정 대상 파일 | 수정 내용 |
|----------|--------------|----------|
| API 이름 해석 실패 | `src/indexer/doc-registry.ts` | NAME_MAP에 별칭 추가 |
| 검색 결과 부정확 | `md/*.md` 또는 BM25 파라미터 | 키워드 보강 / 스코어링 조정 |
| 코드 예제 누락 | `md/*.md` | 해당 언어 코드 블록 추가 |
| 보안 규칙 누락 | `src/tool/*.ts` 또는 `md/*.md` | 도구 설명 / 문서에 보안 안내 추가 |
| 필수 정보 누락 | `md/*.md` | 엔드포인트/파라미터 정보 보완 |

## 9. 기대 효과

- **반복 가능한 QA**: 수동 테스트의 암묵적 기준을 코드화하여 누구나 동일한 품질 검증 가능
- **빠른 보완 사이클**: 실패 → 원인 → 수정 → 재검증을 분 단위로 수행
- **문서 품질 보장**: 코드 예제 누락, 보안 안내 빠짐 등을 자동 감지
- **회귀 방지**: 문서 수정 시 기존 시나리오가 깨지는지 즉시 확인
- **NPM 배포 전 게이트**: `test:qa` 통과를 배포 조건으로 활용 가능
