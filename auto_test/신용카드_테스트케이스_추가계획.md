# 신용카드(CN) 테스트 케이스 추가 계획

## Context

현재 결제 테스트 도구에 휴대폰결제(MC) 6개 케이스만 구현되어 있음. 신용카드(CN) 테스트 케이스 10개를 추가하되, **신용카드는 테스트서버를 제공하지 않으므로** 운영서버(mup.mobilians.co.kr) 전용으로 분리 처리해야 함. 실제 결제가 발생하므로 금액 최소화 + 즉시 취소 안내가 필요.

## 수정 대상 파일 및 변경 내용

### 1. `auto_test/server/config.ts` — 운영서버 호스트 추가 export

현재 `apiHost`만 export하고 있어 CN 전용 운영서버 분기 불가. `prodHost`를 별도로 export.

```typescript
export const config = {
  // ... 기존 필드 유지
  apiHost: raw.mobilpay.use_test ? raw.mobilpay.test_host : raw.mobilpay.prod_host,
  prodHost: raw.mobilpay.prod_host as string,  // ← 추가: CN 전용
  // ...
};
```

### 2. `auto_test/server/test-cases.ts` — CN_CASES 배열 추가

- `TestCase` interface에 `amount` 필드를 확장하여 세금 관련 필드 지원 (CN-09, CN-10용)
  - `amount` 필드를 `number`에서 `number | { total: number; tax?: number; tax_free?: number; supply_value?: number }` 로 변경
- `CN_CASES` 배열 10개 정의
- `ALL_CASES`에 `CN_CASES` 포함

### 3. `auto_test/server/app.ts` — CN 운영서버 분기 + 세금 필드 처리

**거래등록 API 호출 부분 (`/api/test/start`):**
- `tc.cashCode === "CN"`이면 `config.prodHost` 사용, 아니면 기존 `config.apiHost` 사용
- `amount` 필드 처리: TestCase의 amount가 객체이면 그대로 전달, 숫자이면 `{ total: amount }` 변환

**하이브리드 승인 API 호출 부분 (`callApproval`):**
- CN 하이브리드의 경우 `mc_user_key`가 불필요 — 기존 코드에서 `if (tc.cashCode === "MC")` 조건으로 이미 분기되어 있어 추가 변경 불필요

### 4. `auto_test/views/index.html` — UI 변경

**섹션 분리:**
- 기존 하드코딩된 `<div class="section-title">휴대폰결제 (MC) 테스트 케이스</div>` 제거
- `renderCases()` 함수에서 cashCode별로 그룹핑하여 섹션 동적 렌더링:
  - `MC` 그룹: "휴대폰결제 (MC) 테스트 케이스"
  - `CN` 그룹: "신용카드 (CN) 테스트 케이스"

**경고 배너 추가:**
- CN 섹션 상단에 경고 배너:
  - "⚠ 신용카드는 테스트서버를 제공하지 않습니다. 운영서버에서 실제 결제가 발생합니다."
  - "테스트 후 반드시 결제취소를 진행해주세요."
- 빨간/주황 계열 배경으로 시각적 경고

**결제창 사이즈:**
- 이미 index.html 126~129줄에서 CN은 820x600, 나머지는 390x613으로 분기 처리되어 있어 추가 변경 불필요

### 5. `auto_test/결제테스트_사용가이드.md` — 문서 업데이트

**4-2 섹션 (테스트 케이스 확인):**
- CN 10개 케이스 테이블 추가
- "⚠ 신용카드 테스트 주의사항" 별도 블록 추가:
  - 운영서버 전용 (테스트서버 미지원)
  - 실제 결제 발생 → 테스트 후 결제취소 필수
  - config.json의 `use_test` 설정과 무관하게 운영서버 사용

**4-4 섹션 (케이스별 테스트 포인트):**
- CN-01~CN-06 기본 플로우 테스트 포인트 추가
- CN-07~CN-10 전용 옵션 테스트 포인트 추가

## 구현 순서

1. `config.ts` — `prodHost` 추가
2. `test-cases.ts` — `TestCase` interface 수정 + `CN_CASES` 추가 + `ALL_CASES` 업데이트
3. `app.ts` — CN 운영서버 분기 + amount 객체 처리
4. `index.html` — 섹션 그룹핑 + CN 경고 배너
5. `결제테스트_사용가이드.md` — 문서 업데이트

## 검증 방법

1. `cd auto_test && npm run build` — 타입스크립트 컴파일 성공 확인
2. `npm start` 후 `http://localhost:3100` 접속 — MC 6개 + CN 10개 섹션 분리 표시 확인
3. CN 섹션 상단 경고 배너 표시 확인
4. CN 케이스 "테스트 시작" 클릭 시 운영서버(`mup.mobilians.co.kr`)로 거래등록 요청 확인 (서버 콘솔 로그)
5. CN-07~CN-10 케이스의 payOptions가 거래등록 요청에 정상 포함되는지 확인
