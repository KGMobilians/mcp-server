# VS Code + GitHub Copilot에서 MOBILPAY Agent Skill 설정 가이드

## 개요

VS Code에서 GitHub Copilot과 함께 결제 연동 Agent Skill을 사용하려면 세 가지를 설정해야 합니다:

1. **MCP Server** — API 명세 검색/조회 도구 연결
2. **Copilot Instructions** — AI가 결제 연동 코드를 올바르게 작성하도록 가이드하는 지침
3. **Copilot Chat Participant** — `@workspace` 참가자를 통한 프로젝트 컨텍스트 활용 (선택사항)

## 사전 요구사항

- VS Code 최신 버전 (1.99+)
- GitHub Copilot 확장 프로그램 설치 및 구독 활성화
- Node.js 18+ 설치 (MCP 서버 실행용)
- VS Code 설정에서 MCP 기능 활성화 필요:
  - `Cmd+Shift+P` → `Preferences: Open Settings (JSON)`
  - `"chat.mcp.enabled": true` 추가

## 파일 배치 구조

```
프로젝트 루트/
├── .vscode/
│   └── mcp.json                          ← MCP 서버 설정
├── .github/
│   └── copilot-instructions.md           ← Copilot 지침 (Agent Skill)
```

## Step 1: MCP Server 설정

프로젝트 루트에 `.vscode/mcp.json` 파일을 생성합니다.

```json
{
  "servers": {
    "mobilpay": {
      "command": "npx",
      "args": ["-y", "@mobilpay/mcp-server@latest"]
    }
  }
}
```

> 이 파일은 본 가이드 폴더의 `vscode/mcp.json`을 `.vscode/mcp.json`으로 복사하면 됩니다.

**주의:** VS Code의 MCP 설정 형식은 Cursor/Kiro와 다릅니다.

| IDE | 최상위 키 | 파일 위치 |
|-----|----------|----------|
| Cursor | `mcpServers` | `.cursor/mcp.json` |
| Kiro | `mcpServers` | `.kiro/settings/mcp.json` |
| VS Code | `servers` | `.vscode/mcp.json` |

### MCP 서버 동작 확인

1. VS Code 실행 후 `Cmd+Shift+P` → `MCP: List Servers`
2. `mobilpay` 서버가 표시되는지 확인
3. 서버를 시작하고 Copilot Chat에서 MCP 도구 사용 가능 여부 확인

## Step 2: Copilot Instructions (Agent Skill) 설정

프로젝트 루트에 `.github/copilot-instructions.md` 파일을 생성합니다. 이 파일은 GitHub Copilot이 프로젝트 컨텍스트로 참조하는 지침 파일입니다.

> 본 가이드 폴더의 `github/copilot-instructions.md`를 `.github/copilot-instructions.md`로 복사하면 됩니다.

### Copilot Instructions vs 다른 IDE Skill 차이점

| 항목 | Cursor Rules | Kiro Steering | Copilot Instructions |
|------|-------------|---------------|---------------------|
| 파일 형식 | `.mdc` (frontmatter) | `.md` | `.md` |
| 조건부 적용 | 가능 (globs, alwaysApply) | 불가 (항상 로드) | 불가 (항상 로드) |
| 위치 | `.cursor/rules/` | `.kiro/steering/` | `.github/` |
| 파일 수 | 여러 파일 가능 | 여러 파일 가능 | 단일 파일 권장 |

Copilot Instructions는 **단일 파일**(`copilot-instructions.md`)에 모든 지침을 포함하는 것이 권장됩니다. 따라서 MOBILPAY와 내죠여왕 규칙이 하나의 파일에 통합되어 있습니다.

### Copilot Instructions 활성화 확인

1. `Cmd+Shift+P` → `Preferences: Open Settings (UI)`
2. `github.copilot.chat.codeGeneration.useInstructionFiles` 검색
3. 체크박스 활성화 확인

## Step 3: @workspace 참가자 활용 (선택사항)

VS Code Copilot Chat에서 `@workspace` 참가자를 사용하면 프로젝트 파일을 직접 참조할 수 있습니다.

```
@workspace skills/mobilpay-integration/SKILL.md 규칙을 참고해서
MOBILPAY 휴대폰결제 연동 코드를 작성해줘
```

이 방법은 `copilot-instructions.md` 설정 없이도 사용할 수 있지만, 매번 파일 경로를 지정해야 하는 불편함이 있습니다.

## 사용 방법

### Agent 모드에서 사용 (권장)

Copilot Chat에서 모드를 **Agent**로 설정한 후 요청합니다:

```
MOBILPAY 휴대폰결제 연동 코드를 Node.js로 작성해줘
```

**Agent 모드 동작 흐름:**
1. Copilot이 `copilot-instructions.md`의 보안 규칙과 가이드를 컨텍스트로 로드
2. 규칙에 따라 코드 생성 (skey 보호, HMAC 서버사이드 등)
3. 상세 API 명세 필요 시 MCP Tool(`get-payment-api-spec` 등) 호출
4. 체크리스트 기반 코드 검증 후 결과 제공

### 활용 예시 프롬프트

| 요청 | 사용되는 MCP Tool |
|------|------------------|
| "MOBILPAY 거래등록 API 연동해줘" | `get-payment-api-spec` |
| "내죠여왕 결제요청 코드 작성해줘" | `get-nezo-api-spec` |
| "HMAC 검증 코드를 Java로 작성해줘" | `get-payment-code-example` |
| "알림톡 결제 콜백 핸들러 만들어줘" | `get-nezo-api-spec`, `get-nezo-code-example` |
| "결제 관련 문서 검색해줘" | `get-docs` |

## 주의사항

- Copilot Instructions는 **모든 Copilot Chat 세션에 항상 적용**됩니다. 결제와 무관한 작업에서도 로드되므로 파일 크기를 적절히 유지하세요.
- MCP 서버는 VS Code에서 아직 실험적(Experimental) 기능일 수 있습니다. `chat.mcp.enabled` 설정이 필요합니다.
- `.github/copilot-instructions.md`는 저장소에 커밋하면 팀원 모두에게 적용됩니다. 개인용으로만 쓰려면 VS Code User Settings에서 설정하세요.
- VS Code의 MCP 설정에서 최상위 키는 `servers`입니다 (`mcpServers`가 아닌 점에 주의).
