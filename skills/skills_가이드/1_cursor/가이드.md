# Cursor에서 MOBILPAY Agent Skill 설정 가이드

## 개요

Cursor에서 결제 연동 Agent Skill을 사용하려면 두 가지를 설정해야 합니다:

1. **MCP Server** — API 명세 검색/조회 도구 연결
2. **Cursor Rules** — AI가 결제 연동 코드를 올바르게 작성하도록 가이드하는 규칙

## 파일 배치 구조

```
프로젝트 루트/
├── .cursor/
│   ├── mcp.json                          ← MCP 서버 설정
│   └── rules/
│       ├── mobilpay-integration.mdc      ← MOBILPAY 결제연동 규칙
│       └── nezo-integration.mdc          ← 내죠여왕 결제연동 규칙
```

## Step 1: MCP Server 설정

프로젝트 루트에 `.cursor/mcp.json` 파일을 생성합니다.

```json
{
  "mcpServers": {
    "mobilpay": {
      "command": "npx",
      "args": ["-y", "@mobilpay/mcp-server@latest"]
    }
  }
}
```

> 이 파일은 본 가이드 폴더의 `mcp.json`을 `.cursor/mcp.json`으로 복사하면 됩니다.

### MCP 서버 동작 확인

1. Cursor 실행 후 `Cmd+Shift+P` → `MCP: List Servers` 또는 Settings > MCP 탭에서 확인
2. `mobilpay` 서버가 `Running` 상태인지 확인
3. 채팅에서 `get-docs` 등 MCP 도구를 호출할 수 있는지 테스트

## Step 2: Cursor Rules (Agent Skill) 설정

프로젝트 루트에 `.cursor/rules/` 디렉토리를 만들고, `.mdc` 파일을 배치합니다.

> 본 가이드 폴더의 `rules/mobilpay-integration.mdc`와 `rules/nezo-integration.mdc`를 `.cursor/rules/`로 복사하면 됩니다.

### .mdc 파일 구조 설명

```yaml
---
description: MOBILPAY(KG파이낸셜) REST API 결제 연동 코드 작성 시 적용
globs:
alwaysApply: false
---
```

| 필드 | 설정값 | 의미 |
|------|--------|------|
| `description` | 스킬 설명 | Cursor Agent가 언제 이 규칙을 적용할지 판단하는 기준 |
| `globs` | (비워둠) | 특정 파일 패턴에만 적용하려면 설정 (예: `**/*.ts`) |
| `alwaysApply` | `false` | **Agent-Requested** 모드. 항상 적용이 아닌, Agent가 관련 키워드 감지 시 자동 적용 |

### 적용 모드 종류

| 모드 | `alwaysApply` | 동작 |
|------|--------------|------|
| Always | `true` | 모든 채팅에 항상 적용 |
| Agent-Requested | `false` + `globs` 비움 | Agent가 description 기반으로 자동 판단하여 적용 |
| Auto Attached | `false` + `globs` 설정 | 특정 파일 패턴 작업 시 자동 적용 |

현재 설정은 **Agent-Requested** 모드로, "MOBILPAY", "결제 연동", "HMAC" 등의 키워드를 감지하면 자동으로 규칙이 활성화됩니다.

## 사용 방법

Cursor Chat (Agent 모드)에서 다음과 같이 요청합니다:

```
MOBILPAY 휴대폰결제 연동 코드를 Node.js로 작성해줘
```

**자동 동작 흐름:**
1. Agent가 "MOBILPAY", "결제 연동" 키워드 감지
2. `mobilpay-integration.mdc` 규칙 자동 활성화
3. 규칙에 따라 보안 규칙(skey 보호, HMAC 서버사이드 등) 자동 적용
4. 상세 API 명세 필요 시 MCP Tool(`get-payment-api-spec` 등) 자동 호출
5. 체크리스트 기반 코드 검증 후 결과 제공

### 활용 예시 프롬프트

| 요청 | 활성화되는 규칙 | 사용되는 MCP Tool |
|------|----------------|------------------|
| "MOBILPAY 거래등록 API 연동해줘" | mobilpay-integration | `get-payment-api-spec` |
| "내죠여왕 결제요청 코드 작성해줘" | nezo-integration | `get-nezo-api-spec` |
| "결제 취소 기능 구현해줘" | mobilpay-integration | `get-payment-api-spec`, `get-payment-code-example` |
| "HMAC 검증 코드 작성해줘" | mobilpay-integration | `get-payment-api-spec` |
| "알림톡 결제 콜백 핸들러 만들어줘" | nezo-integration | `get-nezo-api-spec`, `get-nezo-code-example` |

## 주의사항

- `.mdc` 파일은 Cursor 전용 형식입니다. 다른 IDE에서는 사용할 수 없습니다.
- `alwaysApply: true`로 변경하면 결제와 무관한 작업에서도 규칙이 적용되어 불필요한 컨텍스트가 추가됩니다. `false`를 권장합니다.
- 규칙이 적용되고 있는지 확인하려면 Cursor Chat에서 응답 하단의 "Rules Applied" 섹션을 확인하세요.
