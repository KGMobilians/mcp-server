# VS Code + Claude Code에서 MOBILPAY Agent Skill 설정 가이드

## 개요

VS Code에서 Claude Code 확장(또는 Claude Code CLI)과 함께 사용하는 경우, **추가 설정이 필요 없습니다.** Claude Code는 `skills/` 디렉토리의 `SKILL.md` 파일을 네이티브로 지원합니다.

## 왜 추가 설정이 불필요한가

Claude Code는 다음을 자동으로 인식합니다:

| 항목 | 파일 | 동작 |
|------|------|------|
| MCP 서버 설정 | `CLAUDE.md` | 프로젝트 설명에 MCP 도구 정보 포함 |
| MOBILPAY Skill | `skills/mobilpay-integration/SKILL.md` | 자동 인식, 결제 연동 요청 시 활용 |
| NEZO Skill | `skills/nezo-integration/SKILL.md` | 자동 인식, 알림톡 결제 요청 시 활용 |
| 레퍼런스 문서 | `skills/*/references/**/*.md` | Skill에서 참조 시 자동 로드 |
| 평가 데이터 | `skills/*/evals/evals.json` | Skill 품질 평가에 활용 |

## 이미 완료된 설정

### CLAUDE.md (프로젝트 루트)

`CLAUDE.md`는 Claude Code의 프로젝트 지침 파일로, 이미 다음 정보를 포함하고 있습니다:
- MCP 서버의 6개 도구 설명
- 도메인 보안 규칙 (MOBILPAY + NEZO)
- 프로젝트 아키텍처 및 문서 구조

### Skills 디렉토리 구조

```
skills/
├── mobilpay-integration/
│   ├── SKILL.md              ← Skill 정의 (자동 인식)
│   ├── evals/
│   │   └── evals.json        ← 평가 데이터
│   └── references/            ← 레퍼런스 문서
│       ├── api/               ← API 명세 (11 files)
│       ├── guides/            ← 연동 가이드 (5 files)
│       └── reference/         ← 코드표/에러코드 (2 files)
└── nezo-integration/
    ├── SKILL.md              ← Skill 정의 (자동 인식)
    ├── evals/
    │   └── evals.json        ← 평가 데이터
    └── references/            ← 레퍼런스 문서
        ├── api/               ← API 명세 (5 files)
        ├── guides/            ← 연동 가이드 (2 files)
        └── reference/         ← 레퍼런스 (2 files)
```

### SKILL.md 파일 구조

Claude Code의 Skill은 frontmatter와 본문으로 구성됩니다:

```markdown
---
name: mobilpay-integration
description: >
  MOBILPAY(KG파이낸셜) REST API 결제 연동 전문가 스킬. ...
  이 스킬은 사용자가 MOBILPAY, KG파이낸셜 등을 언급하거나 ...
---

# MOBILPAY 결제 연동 스킬

(본문: 결제 플로우, 보안 규칙, 체크리스트 등)
```

| frontmatter 필드 | 역할 |
|------------------|------|
| `name` | 스킬 고유 이름 |
| `description` | 스킬 설명 + AI가 이 스킬을 언제 사용할지 판단하는 기준 (트리거 키워드 포함) |

## 사용 방법

### Claude Code CLI에서

터미널에서 Claude Code를 실행하고 요청합니다:

```bash
claude
```

```
> MOBILPAY 휴대폰결제 연동 코드를 Node.js로 작성해줘
```

### VS Code Claude Code 확장에서

VS Code에서 Claude Code 패널을 열고 요청합니다:

```
MOBILPAY 거래등록 API 연동 코드를 작성해줘
```

### 동작 흐름

1. Claude Code가 요청에서 "MOBILPAY", "결제 연동" 등 키워드 감지
2. `skills/mobilpay-integration/SKILL.md`의 `description` 매칭 → Skill 활성화
3. SKILL.md 본문의 보안 규칙, 플로우, 체크리스트를 컨텍스트로 로드
4. 상세 API 명세 필요 시 `references/` 디렉토리의 문서 참조
5. MCP 서버 도구(`get-payment-api-spec` 등)도 필요 시 호출 가능
6. 체크리스트 기반 코드 검증 후 결과 제공

## 다른 IDE와의 비교

| 항목 | Claude Code | Cursor | Kiro | VS Code Copilot |
|------|------------|--------|------|-----------------|
| Skill 형식 | `SKILL.md` (frontmatter) | `.mdc` (frontmatter) | `.md` (plain) | `.md` (plain) |
| Skill 위치 | `skills/*/SKILL.md` | `.cursor/rules/` | `.kiro/steering/` | `.github/` |
| 조건부 적용 | description 기반 자동 | globs, alwaysApply | 항상 로드 | 항상 로드 |
| Reference 지원 | `references/` 자동 참조 | 불가 (MCP로 대체) | 불가 (MCP로 대체) | 불가 (MCP로 대체) |
| 평가 데이터 | `evals/` 지원 | 미지원 | 미지원 | 미지원 |
| 추가 설정 필요 | 없음 | `.mdc` 파일 변환 필요 | steering 파일 변환 필요 | instructions 파일 변환 필요 |

### Claude Code의 장점

1. **Reference 직접 참조**: `references/` 디렉토리의 상세 API 문서를 직접 읽을 수 있어, MCP 서버 없이도 정확한 코드 생성 가능
2. **변환 불필요**: 다른 IDE는 SKILL.md를 해당 IDE 형식으로 변환해야 하지만, Claude Code는 원본을 그대로 사용
3. **평가 지원**: `evals/evals.json`으로 Skill의 코드 생성 품질을 측정하고 개선 가능

## 주의사항

- `skills/` 디렉토리는 프로젝트 루트에 위치해야 합니다.
- `SKILL.md`의 `description` 필드에 트리거 키워드가 충분히 포함되어 있어야 자동 매칭이 잘 됩니다.
- `references/` 디렉토리 내 문서는 SKILL.md에서 상대 경로로 참조합니다 (예: `references/api/registration.md`).
