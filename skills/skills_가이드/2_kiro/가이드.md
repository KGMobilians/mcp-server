# Kiro에서 MOBILPAY Agent Skill 설정 가이드

## 개요

Kiro(AWS의 AI IDE)에서 결제 연동 Agent Skill을 사용하려면 두 가지를 설정해야 합니다:

1. **MCP Server** — API 명세 검색/조회 도구 연결
2. **Steering Files** — AI가 결제 연동 코드를 올바르게 작성하도록 가이드하는 문서

## 파일 배치 구조

```
프로젝트 루트/
├── .kiro/
│   ├── settings/
│   │   └── mcp.json                         ← MCP 서버 설정
│   └── steering/
│       ├── mobilpay-integration.md           ← MOBILPAY 결제연동 가이드
│       └── nezo-integration.md               ← 내죠여왕 결제연동 가이드
```

## Step 1: MCP Server 설정

프로젝트 루트에 `.kiro/settings/mcp.json` 파일을 생성합니다.

```json
{
  "mcpServers": {
    "mobilpay": {
      "command": "npx",
      "args": ["-y", "@mobilpay/mcp-server@latest"]
    }
  }
}
```

> 이 파일은 본 가이드 폴더의 `settings/mcp.json`을 `.kiro/settings/mcp.json`으로 복사하면 됩니다.

### MCP 서버 동작 확인

1. Kiro 실행 후 좌측 사이드바에서 MCP 섹션 확인
2. `mobilpay` 서버가 활성화 상태인지 확인
3. 채팅에서 MCP 도구를 호출할 수 있는지 테스트

## Step 2: Steering Files (Agent Skill) 설정

Kiro에서 Skill에 해당하는 기능은 **Steering Files**입니다. `.kiro/steering/` 디렉토리에 마크다운 파일을 배치하면 Kiro의 AI Agent가 해당 내용을 프로젝트 컨텍스트로 인식합니다.

> 본 가이드 폴더의 `steering/mobilpay-integration.md`와 `steering/nezo-integration.md`를 `.kiro/steering/`으로 복사하면 됩니다.

### Steering 파일 vs Cursor Rules 차이점

| 항목 | Cursor Rules (`.mdc`) | Kiro Steering (`.md`) |
|------|----------------------|----------------------|
| 파일 형식 | YAML frontmatter + Markdown | 일반 Markdown |
| 조건부 적용 | `alwaysApply`, `globs`로 제어 | 항상 컨텍스트로 제공 |
| 위치 | `.cursor/rules/` | `.kiro/steering/` |

Kiro Steering 파일은 별도의 frontmatter가 필요 없으며, `.kiro/steering/` 디렉토리에 배치하면 자동으로 프로젝트 컨텍스트에 포함됩니다.

## Step 3: Kiro Hooks 활용 (선택사항)

Kiro의 **Hooks**는 이벤트 기반 자동화 기능으로, 결제 관련 코드 작성 시 자동으로 보안 검증을 수행하도록 설정할 수 있습니다.

### Hooks 설정 방법

Kiro에서 `Cmd+Shift+P` → `Kiro: Create Hook`을 실행하거나, `.kiro/hooks/` 디렉토리에 직접 파일을 생성합니다.

### 활용 예시: 결제 코드 보안 체크

```markdown
# 결제 코드 보안 검증 Hook

## Trigger
결제 관련 파일이 저장될 때

## Checks
1. skey/MAC_KEY가 하드코딩되어 있지 않은지 확인
2. HMAC/MAC 검증이 서버사이드에서만 수행되는지 확인
3. noti_url/callback_url 핸들러에 멱등성 로직이 있는지 확인
4. 결제 API 호출이 백엔드에서만 이루어지는지 확인
```

> Hooks는 선택사항입니다. Steering 파일만으로도 AI가 규칙을 인지하고 코드를 생성합니다.

## 사용 방법

Kiro Chat에서 다음과 같이 요청합니다:

```
MOBILPAY 휴대폰결제 연동 코드를 Node.js로 작성해줘
```

**자동 동작 흐름:**
1. Kiro Agent가 Steering 파일의 MOBILPAY 가이드를 컨텍스트로 로드
2. 보안 규칙, API 엔드포인트, 결제 플로우 정보를 기반으로 코드 생성
3. 상세 API 명세 필요 시 MCP Tool 호출
4. 체크리스트 기반 코드 검증

### 활용 예시 프롬프트

| 요청 | 참조되는 Steering 파일 | 사용되는 MCP Tool |
|------|----------------------|------------------|
| "MOBILPAY 거래등록 API 연동해줘" | mobilpay-integration.md | `get-payment-api-spec` |
| "내죠여왕 결제요청 코드 작성해줘" | nezo-integration.md | `get-nezo-api-spec` |
| "결제 취소 기능 구현해줘" | mobilpay-integration.md | `get-payment-api-spec` |
| "알림톡 결제 콜백 핸들러 만들어줘" | nezo-integration.md | `get-nezo-api-spec` |

## Kiro Specs 활용 (고급)

Kiro의 **Specs** 기능을 사용하면 결제 연동을 구조화된 요구사항으로 정의할 수 있습니다.

```
"MOBILPAY 결제 연동 구현" Spec을 만들어줘
```

Kiro가 자동으로 Requirements → Design → Implementation 단계로 분리하여 체계적으로 결제 연동을 진행합니다. Steering 파일의 보안 규칙이 각 단계에 자동으로 반영됩니다.

## 주의사항

- Steering 파일은 항상 컨텍스트로 제공되므로, 결제와 무관한 작업에서도 로드됩니다. 파일 크기가 너무 크면 성능에 영향을 줄 수 있으니 핵심 규칙 위주로 유지하세요.
- Kiro는 `.cursor/rules/*.mdc` 파일을 인식하지 않습니다. 반드시 `.kiro/steering/` 디렉토리에 별도 파일을 배치해야 합니다.
- MCP 서버 설정도 `.kiro/settings/mcp.json`에 있어야 합니다 (`.cursor/mcp.json`과 별도).
