# @mobilpay/mcp-server npm 패키지 보안 강화 계획

## Context

npmjs.com의 `@mobilpay/mcp-server` 패키지 설정에서 다음 보안 취약점이 발견됨:
- Trusted Publisher 미설정 → supply chain attack 취약
- Publishing access가 권장 수준보다 낮음 → 토큰 유출 시 배포 가능
- 단일 maintainer → bus factor 1
- CI/CD 파이프라인 없음 → 수동 배포만 가능

**목표:** GitHub Actions 기반 자동화된 안전한 배포 파이프라인 구축 및 npm 보안 설정 강화

---

## Part 1: 코드 변경 (Claude가 실행)

### 1-1. `.gitignore` 생성

```
node_modules/
dist/
.DS_Store
*.tgz
```

> `md/` 디렉토리는 런타임과 테스트에 필요하므로 gitignore에 포함하지 않음

### 1-2. `.github/workflows/ci.yml` 생성

CI 파이프라인: `main` 브랜치 push 및 PR에서 빌드+테스트 실행

```yaml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]
jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
      - run: npm ci
      - run: npm run build
      - run: npm test
```

### 1-3. `.github/workflows/publish.yml` 생성

OIDC Trusted Publisher 기반 npm 배포 (npm 토큰 불필요)

```yaml
name: Publish to npm
on:
  release:
    types: [published]
jobs:
  publish:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      id-token: write
    environment: npm-publish
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'
          registry-url: 'https://registry.npmjs.org'
      - run: npm ci
      - run: npm run build
      - run: npm test
      - run: npm publish --provenance --access public
```

핵심 포인트:
- `id-token: write` — OIDC 토큰 발급에 필요
- `environment: npm-publish` — npmjs.com Trusted Publisher 등록 시 사용할 환경 이름
- `--provenance` — SLSA 출처 증명 첨부
- `--access public` — 스코프 패키지 공개 배포에 필요
- release 이벤트 트리거 → GitHub Release 생성 시 자동 배포

## Part 2: 수동 조치 (사용자가 직접 실행, 순서 중요)

> ⚠️ 반드시 아래 순서대로 진행해야 합니다. Trusted Publisher 설정 전에 토큰을 비활성화하면 자동 배포가 불가능해집니다.

### Step 1: GitHub 리포지토리 생성 및 코드 push

```bash
cd /Users/finalgod/windsurf_workspace/mobilpay_mcp
git init
git add .
git commit -m "Initial commit"
git remote add origin <GitHub 리포지토리 URL>
git push -u origin main
```

### Step 2: GitHub Environment 생성

1. GitHub 리포지토리 → Settings → Environments
2. "New environment" 클릭
3. 이름: `npm-publish` (정확히 일치해야 함)
4. (선택) "Required reviewers" 보호 규칙 추가 → 배포 전 승인 필요

### Step 3: npmjs.com에 Trusted Publisher 등록

1. npmjs.com에 `mobil_aitf`로 로그인
2. `@mobilpay/mcp-server` 패키지 Settings 이동
3. Trusted Publishers 섹션에서 "GitHub Actions" 선택
4. 입력:
   - Repository owner: `<GitHub org/user>`
   - Repository name: `<repo name>`
   - Workflow filename: `publish.yml`
   - Environment: `npm-publish`
5. 저장

### Step 4: 배포 파이프라인 테스트

1. `package.json`과 `src/index.ts`의 version을 `2.0.2`로 변경
2. commit & push
3. GitHub에서 Release 생성 (tag: `v2.0.2`)
4. Actions 탭에서 publish workflow 성공 확인
5. npmjs.com에서 provenance 뱃지 확인

### Step 5: Publishing Access 강화

**테스트 배포 성공 후에만 진행**

1. npmjs.com → `@mobilpay/mcp-server` → Settings
2. Publishing access를 **"Require two-factor authentication and disallow tokens (recommended)"** 로 변경
3. 저장

### Step 6: 백업 Maintainer 추가

```bash
npm owner add <npm-username> @mobilpay/mcp-server
```

- 추가할 사용자는 반드시 2FA가 활성화된 npm 계정이어야 함

### Step 7: developers team 멤버 감사

1. npmjs.com → Settings → Teams → `developers`
2. 불필요한 멤버 제거
3. 팀 권한이 `@mobilpay/mcp-server`에 대해 read-write인지 확인

## 검증 방법

1. **CI workflow 검증**: PR 생성 시 빌드+테스트가 자동 실행되는지 확인
2. **Publish workflow 검증**: GitHub Release 생성 → npm에 provenance 포함된 패키지 배포 확인
3. **보안 설정 검증**: `npm audit signatures` 실행하여 서명 확인
4. **Publishing access 검증**: 로컬에서 `npm publish` 시도 → 토큰 방식 차단 확인
